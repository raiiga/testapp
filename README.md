## Testapp

### Опис завдання:

>Модель:
>Компанії-Персони-Завдання
>
>Зв'язки у всіх випадках many-to-many
>У зв'язку є атрібути:
>Компанії-Персони: позиція
>Персони-Завдання: тип участі
>
>Необхідно:
>
>CRUD для всіх типов зв'язків та сутностей
>
>Також:
>Пошук самих "багатокористувацьких" завдань
>Пошук самих "навантажених" персон
>Отримання ієрархії від вказанної компанії до вказанних завдань
>
>Усі три сервіси мають як аргумент також опціональну фільтрацію по атрібутам зв'язку
>
>БД:
>мінімум - SQL(на вибір виконавця)
>бажано - графова
>максимум -  Neo4j

### 🚀 Як запустити:

```bash
# 1. Clone the repository
git clone https://github.com/raiiga/testapp

# 2. Navigate to the project directory
cd testapp/docker

# 3. Apply env variables, manualy or via .env file
export NEO4J_AUTH=...
export NEO4J_USERNAME=...
export NEO4J_PASSWORD=...

# 4. Launch the services using Docker Compose
docker compose up -d

# 5. Open the browser and navigate to http://localhost:8080/swagger-ui/index.html
```

### Задум

Інфраструктура апки складена на основі трьох контейнерів Caddy, Neo4j та Amazon Corretto.
На початку я планував зробити мало-мальський фронтенд і сервити його через Caddy, 
однак робота з графами виявилась набагато цікавішою і в мене лишилось не так багато часу на інше, 
тож на Caddy немає статики він просто віддає трафік на бекенд, а клієнтську частину замінює Swagger.

В якості БД я обрав Neo4j. Мені було важливо взяти стек найближчий до реального проєкту і заглибитись в деталі цією БД:
робота з індексами, внутрішніми та зовнішніми ідентифікаторами, датами, повнотекстовим пошуком, 
роботою із запитами і зв'язками на рівні SDN, збіркою запитів через Cypher DSL, тощо.

Сама апка у вигляді fat-jar працює на базі контейнера Amazon Corretto з 21 версією Java.

Згідно із завдданням я створив сутності 1) Компанії  2) Персони та 3) Завдання, а також зв'язки
з відповідними атрибутами. Окрім атрибутів вказаних в ТЗ, я додав діапазон дат на зв'язки, спочатку 
просто з цікавості спробувати запити на пошук історичних зв'язків, але в підсумку лишив їх і в кінцевому варіанті.
Видалення реалізовано в двох варіантах як через проставлення дати until на зв'язках так і через пряме видалення нод та зв'язків.
